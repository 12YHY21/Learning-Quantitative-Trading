## 1. 回测究竟在模拟什么

回测输入历史可见数据、策略规则、执行假设和初始资金，输出历史条件下的账户变化。

```mermaid
flowchart LR
    D["历史数据"] --> F["特征与信号"]
    F --> W["目标/实际权重"]
    W --> R["策略收益"]
    R --> C["扣除成本"]
    C --> N["净值与风险"]
```

最简向量化回测不逐笔创建订单，而是假设每天持有某个权重。

> [!IMPORTANT] 量化重点
> 首先验证“今天赚的是哪一段收益、用的是哪一天决定的仓位”。收益对齐错一行，漂亮结果可能全部失效。

## 2. 最关键的时间对齐

资产从 $t-1$ 收盘到 $t$ 收盘的简单收益：

$$
r_t=\frac{P_t}{P_{t-1}}-1
$$

若仓位在 $t-1$ 时已经确定，则策略在第 $t$ 日的毛收益为：

$$
r^{gross}_{p,t}=w_{t-1}r_t
$$

因此用 $t$ 日收盘数据生成的目标仓位，至少需要滞后一档：

```python
data["asset_return"] = data["close"].pct_change()
data["position"] = data["signal"].shift(1).fillna(0.0)
data["gross_return"] = data["position"] * data["asset_return"]
```

错误写法：

```python
#错误：本日收盘信号获得了本日此前已经发生的收益
data["gross_return"] = data["signal"] * data["asset_return"]
```

### 时间线

```text
t-1 日收盘：观察 close[t-1]，生成 signal[t-1]
t 日持有区间：position[t] = signal[t-1]
t 日收盘：获得 return[t]
```

这只是收盘到收盘的教学约定。若使用下一日开盘成交，应使用开盘价和更精细的收益区间。

## 3. 成本与换手

单资产仓位变化：

$$
turnover_t=|w_t-w_{t-1}|
$$

若单边综合成本率为 $c$：

$$
cost_t=turnover_t\times c
$$

净收益：

$$
r^{net}_{p,t}=r^{gross}_{p,t}-cost_t
$$

```python
data["turnover"] = data["position"].diff().abs()
data.loc[data.index[0], "turnover"] = abs(data.loc[data.index[0], "position"])
data["cost"] = data["turnover"] * cost_rate
data["net_return"] = data["gross_return"] - data["cost"]
```

这里把多种费用和滑点合成一个比例，只适合第一轮研究。真实 A 股卖出税费、佣金最低收费、过户费和冲击成本可能不是完全对称线性。

> [!IMPORTANT] A 股规则
> 税费、收费标准和适用证券可能调整。正式研究应按交易方向、证券类型、市场、日期和官方核验来源配置。

## 4. 手算五日小样本

假设价格和信号如下：

| 日 | 收盘价 | 收盘后信号 | 次日使用仓位 |
|---|---:|---:|---:|
| 1 | 10.00 | 0 | 0 |
| 2 | 10.50 | 1 | 0 |
| 3 | 10.71 | 1 | 1 |
| 4 | 10.50 | 0 | 1 |
| 5 | 10.60 | 0 | 0 |

第 3 日资产收益：

$$
r_3=\frac{10.71}{10.50}-1=2\%
$$

仓位来自第 2 日信号，所以第 3 日毛策略收益为 2%。

第 4 日：

$$
r_4=\frac{10.50}{10.71}-1\approx-1.9608\%
$$

第 4 日仍持仓，策略承受该损失。第 4 日收盘信号变为 0，第 5 日才空仓。

若忽略成本，期末净值：

$$
1\times1.02\times(1-0.019608)\approx1.00
$$

价格先涨后跌，策略约回到起点。

## 5. 完整单资产实现

```python
from dataclasses import dataclass
import numpy as np
import pandas as pd

@dataclass(frozen=True)
class VectorBacktestResult:
    frame: pd.DataFrame
    total_return: float
    max_drawdown: float

def vector_backtest(
    prices: pd.Series,
    signal: pd.Series,
    cost_rate: float = 0.0,
) -> VectorBacktestResult:
    if cost_rate < 0:
        raise ValueError("成本率不能为负")
    if not prices.index.equals(signal.index):
        raise ValueError("价格与信号索引必须完全一致")
    if prices.isna().any() or (prices <= 0).any():
        raise ValueError("价格必须非空且大于 0")

    frame = pd.DataFrame({
        "close": prices.astype(float),
        "signal": signal.astype(float),
    })
    frame["asset_return"] = frame["close"].pct_change().fillna(0.0)
    frame["position"] = frame["signal"].shift(1).fillna(0.0)
    frame["gross_return"] = frame["position"] * frame["asset_return"]
    frame["turnover"] = frame["position"].diff().abs()
    frame.loc[frame.index[0], "turnover"] = abs(frame.iloc[0]["position"])
    frame["cost"] = frame["turnover"] * cost_rate
    frame["net_return"] = frame["gross_return"] - frame["cost"]
    frame["equity"] = (1.0 + frame["net_return"]).cumprod()
    frame["running_max"] = frame["equity"].cummax()
    frame["drawdown"] = frame["equity"] / frame["running_max"] - 1.0

    return VectorBacktestResult(
        frame=frame,
        total_return=float(frame["equity"].iloc[-1] - 1.0),
        max_drawdown=float(frame["drawdown"].min()),
    )

index = pd.date_range("2026-01-05", periods=5, freq="B")
prices = pd.Series([10.00, 10.50, 10.71, 10.50, 10.60], index=index)
signal = pd.Series([0, 1, 1, 0, 0], index=index)

result = vector_backtest(prices, signal, cost_rate=0.001)
print(result.frame.round(6))
print("总收益:", round(result.total_return, 6))
print("最大回撤:", round(result.max_drawdown, 6))
```

### 净值和回撤

从初始净值 1 开始：

$$
NAV_t=\prod_{k=1}^{t}(1+r^{net}_{p,k})
$$

回撤：

$$
DD_t=\frac{NAV_t}{\max_{1\le k\le t}NAV_k}-1
$$

最大回撤为：

$$
MDD=\min_t DD_t
$$

最大回撤通常是负数；报告时也有人展示其绝对值，必须注明口径。

## 6. 多资产扩展

多资产组合毛收益：

$$
r_{p,t}^{gross}=\sum_{i=1}^{N}w_{i,t-1}r_{i,t}
$$

组合换手的一种定义：

$$
turnover_t=\sum_{i=1}^{N}|w_{i,t}-w_{i,t-1}|
$$

```python
#returns 和 target_weights 的行是日期，列是证券
positions = target_weights.shift(1).fillna(0.0)
gross_return = (positions * returns).sum(axis=1)
turnover = positions.diff().abs().sum(axis=1)
turnover.iloc[0] = positions.iloc[0].abs().sum()
net_return = gross_return - turnover * cost_rate
equity = (1 + net_return).cumprod()
```

有些机构把双边换手除以 2。不同口径会影响数值，报告中必须写公式。

### 权重检查

```python
if (positions.abs().sum(axis=1) > 1.000001).any():
    raise ValueError("检测到超过约定上限的总仓位")
```

若允许杠杆，应明确资金成本和保证金规则。

## 7. 三层递进验证

为定位现实约束的影响，按三层运行：

1. 无成本、无交易限制：只验证时间对齐和数学正确性。
2. 加费用与滑点：观察高换手是否失效。
3. 加 A 股约束：观察无法成交、延迟成交和可卖数量。

输出对照表：

| 场景 | 总收益 | 最大回撤 | 换手 | 拒绝订单 |
|---|---:|---:|---:|---:|
| 无成本 | 待计算 | 待计算 | 待计算 | 不适用 |
| 加成本 | 待计算 | 待计算 | 待计算 | 不适用 |
| 加约束 | 待计算 | 待计算 | 待计算 | 待计算 |

如果加成本后表现骤降，说明策略依赖频繁交易；加约束后下降则可能依赖不可成交的理想价格。

## 8. 常见错误与排错

### 净值第一天不是 1

检查首日收益、首日仓位和首日换手如何定义。建议明确初始化并保留中间列。

### 使用 `cumprod(return)`

错误。应计算 `(1 + return).cumprod()`。

### 价格和信号索引自动错位

Pandas 会按标签对齐。运算前用 `index.equals` 检查，并避免无意的重复索引。

### 成本只有开仓没有平仓

检查换手是否对仓位差的绝对值计算，平仓从 1 到 0 同样产生换手。

### 回测异常平滑

可能把缺失收益填成 0、使用了平滑后的复权价，或错误前向填充停牌期。

## 9. 向量化模型的边界

它适合：

- 日频、成交假设简单的快速筛选。
- 检查收益对齐与成本敏感性。
- 因子分组和大量参数的第一轮研究。

它不擅长：

- 部分成交和订单有效期。
- 同日多个事件的先后顺序。
- T+1 可卖数量。
- 停牌、涨跌停导致订单延后。
- 逐笔现金账本和撤单。

> [!TIP] 工程验收
> - 小样本程序结果与手算一致。
> - 所有中间列可检查，不只返回一个收益率。
> - 零信号时毛收益为零。
> - 成本为零时净收益等于毛收益。
> - 加正成本后，其他条件不变时净值不会高于无成本结果。

## 本章总结

向量化回测的核心是清晰的时间对齐：过去决定仓位，仓位获得之后的收益，换手产生成本。它是研究的第一把标尺，而不是完整交易所模拟器。

## 贯穿案例检查点：三个最小性质测试

```python
def test_no_position_means_no_gross_return():
    prices = pd.Series([10.0, 11.0, 9.0])
    signal = pd.Series([0, 0, 0])
    result = vector_backtest(prices, signal)
    assert (result.frame["gross_return"] == 0).all()

def test_positive_cost_never_improves_same_path():
    prices = pd.Series([10.0, 11.0, 10.0])
    signal = pd.Series([1, 1, 0])
    free = vector_backtest(prices, signal, 0.0)
    paid = vector_backtest(prices, signal, 0.001)
    assert paid.frame["equity"].iloc[-1] <= free.frame["equity"].iloc[-1]

def test_signal_is_delayed():
    prices = pd.Series([10.0, 11.0])
    signal = pd.Series([1, 0])
    result = vector_backtest(prices, signal)
    assert result.frame["position"].tolist() == [0.0, 1.0]
```

这些是“性质测试”：不依赖策略赚钱，而检查实现永远应满足的规律。

> [!TIP] 工程验收
> 修改成本、指标或数据代码后，以上性质测试仍通过，防止重构悄悄改变时间对齐。
